# pos_billing

This repository is the central workspace for the POS Billing System.
All major components of the platform—Frontend, Backend, and QA Automation—are maintained here in a single repository to keep development aligned, reduce version mismatch, and simplify coordination between teams.

This structure allows teams to work independently within their domains while still moving forward as one system.

---

## Development Workflow (Standard Operating Procedure)

### Branching Rules

- No direct commits to main
- All work must be done in a separate branch created for a specific task
- Branch names must clearly describe the purpose of the change

---

### Pull Request & Review Process

- All changes must be submitted via a Pull Request targeting main
- Relevant Code Owners are automatically assigned for review
- Approval from the respective Code Owner is mandatory before merge

---

### Merge Conflicts

- Communicate before resolving conflicts involving other domains
- Resolve conflicts locally and update the Pull Request

---

## Contribution Guidelines

- Stay within your assigned domain
- Write clear, professional commit messages
- Update domain-specific README files when making major changes
